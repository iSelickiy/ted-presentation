# MyOwnVPN — Гайд по развёртыванию собственного VPN

Этот гайд проведёт тебя от нуля до работающего персонального VPN, который сам
решает, через какую страну направить трафик. Российские сервисы идут напрямую,
зарубежные — через второй сервер. Один профиль в приложении, без ручного
переключения.

**На кого рассчитан:** на обычного пользователя, который умеет открыть терминал
и скопировать команду. Всё остальное сделает AI-агент.

---

## Что ты получишь

- Один VPN-профиль в приложении **AmneziaVPN** (iPhone / Mac / Android / Windows).
- Российские сервисы (банки, госуслуги, Яндекс, ВК, маркетплейсы) работают
  напрямую через российский сервер.
- Зарубежные сервисы (YouTube, Instagram, ChatGPT, OpenAI) уходят через второй
  сервер за границей.
- Никаких ручных переключений между профилями. Всё работает в одном подключении.
- Возможность выдать доступ членам семьи — у каждого свой конфиг.

## Что понадобится

### Железо и софт

| Что | Зачем |
|-----|-------|
| Компьютер (Mac / Windows / Linux) | С него будешь запускать AI-агента |
| Claude Code или OpenAI Codex CLI | AI-агент, который всё развернёт |
| 2 VPS-сервера | На них будет работать VPN |

### Серверы

Нужно два сервера. Один в России (или рядом), один за границей.

**Минимальные требования к каждому:**
- Ubuntu 24.04 x86_64
- 1 vCPU, 1 GB RAM (можно и 512 MB, но 1 GB комфортнее)
- Публичный IPv4-адрес
- Доступ по SSH под пользователем `root` (по паролю или SSH-ключу)
- Открытые порты в файрволе провайдера:
  - `22/tcp` (SSH)
  - `443/tcp` (Reality — вход для клиентов)
  - `51820/udp` (WireGuard — туннель между серверами)

**Где купить:** любой VPS-провайдер. Важно:
- Российский сервер — любой хостинг в РФ (Selectel, TimeWeb, FirstVDS, VDSina и т.д.)
- Зарубежный сервер — любой хостинг не в РФ (Hetzner, DigitalOcean, Vultr, Aeza и т.д.)

После покупки зайди в панель управления сервером и убедись, что порты `443/tcp`
и `51820/udp` открыты. Обычно это раздел «Firewall» или «Security Groups».

## Шаг 1: Собери данные о серверах

Подключись к каждому серверу по SSH (провайдер даст IP, логин и пароль) и выполни
по одной команде:

```bash
curl -4 ifconfig.me && echo && ip -4 addr show | grep -E 'inet .* (eth|ens|enp)' | awk '{print $2, $NF}'
```

Пример вывода:
```
203.0.113.10
203.0.113.10/24 eth0
```

## Шаг 5: Что произойдёт дальше

Агент выполнит автоматически:

1. Создаст все файлы проекта (скрипты, конфиги, правила маршрутизации).
2. Сгенерирует криптографические ключи (WireGuard + Reality).
3. Проверит доступность серверов по SSH.
4. Установит нужные пакеты на оба сервера (WireGuard, nftables, sing-box).
5. Сгенерирует и загрузит конфигурации на серверы.
6. Запустит все сервисы.
7. Выполнит проверки здоровья (`health_check` и `smoke_check`).
8. Выдаст готовые клиентские конфиги.

**Общее время:** 10-15 минут.

## Шаг 6: Импортируй конфиг в AmneziaVPN

1. Скачай **AmneziaVPN** из App Store (iPhone/Mac) или с [официального сайта](https://amnezia.org) (Windows/Android/Linux).
2. Открой приложение.
3. Нажми **«Добавить из файла»** (или «Import from file»).
4. Выбери файл из папки проекта: `build/clients/твоё-имя.xray.json`
5. Нажми «Подключиться».

Готово. VPN работает.

## Как добавлять новых пользователей

Через 2 недели работы решил выдать доступ жене? Всё просто.

Открой терминал в папке проекта и скажи агенту:

> Добавь пользователя: имя «wife-iphone», отображаемое имя «Жена iPhone», платформа ios. Запусти деплой.

Агент сам:
1. Добавит запись в `config/users.yaml`.
2. Сгенерирует конфиг.
3. Задеплоит на серверы.
4. Отдаст файл `build/clients/wife-iphone.xray.json` — останется только импортировать в AmneziaVPN на телефоне жены.

Чтобы отключить пользователя:

> Отключи пользователя wife-iphone и задеплой.

## Опционально: Telegram-бот

Бот — это интерфейс управления VPN-ключами прямо в Telegram. Больше не нужно
открывать терминал, чтобы добавить пользователя или скачать конфиг.

### Что умеет бот

| Функция | Как работает |
|---------|-------------|
| Авторизация | Через «Поделиться контактом» — бот проверяет номер телефона по ACL |
| Создать ключ | Пользователь нажимает «Новый ключ», вводит имя, выбирает платформу (iOS/Android/Mac/Windows) |
| Скачать конфиг | Кнопка «📥 Xray JSON» — файл для AmneziaVPN. Кнопка «📱 VLESS + QR» — ссылка + QR-код |
| Отозвать ключ | Кнопка «Отозвать» — конфиг перестаёт приниматься сервером |
| Обновить ключ | Кнопка «Обновить» — старый отзывается, создаётся новый (без перерыва в доступе) |
| Деплой | Автоматический render + apply после каждого изменения ключей |
| Аудит | Все действия пишутся в лог: кто, когда, какой ключ создал/отозвал |

### Роли в боте

- **Админ** (`admins`) — безлимитные ключи, видит все ключи всех пользователей, может запускать деплой
- **Пользователь** (`users`) — до 5 ключей по умолчанию, видит только свои ключи
- Можно пригласить друга по invite-коду (через команду `/code <код>`)

### Как деплой работает в боте

1. Пользователь создаёт/отзывает ключ → запись попадает в очередь деплоя
2. Фоновый процесс бота ждёт 5-30 секунд (батчинг: несколько правок → один деплой)
3. Бот запускает `render_artifacts.py --strict && bootstrap_cluster.sh --apply-only`
4. После успешного деплоя бот обновляет SQLite-кеш и уведомляет админа
5. Если деплой упал — бот сообщает админу об ошибке

### Архитектура бота

Бот работает на **RU-сервере** как systemd-сервис `myownvpn-bot.service`. Код лежит
в `/opt/myownvpn/bot/`. Бот использует тот же `render_artifacts.py` и
`bootstrap_cluster.sh`, что и ручной деплой.

Бот **не заменяет** существующий pipeline, а автоматизирует его:
- Source of truth — `config/users.yaml`
- SQLite — кеш, очередь деплоя и аудит-лог
- YAML-файл защищён `fcntl`-локом от одновременной записи

### Сеть бота (SOCKS5-прокси)

Бот работает на RU-сервере, где Telegram API может быть заблокирован. Для доступа
к Telegram API бот использует SOCKS5-прокси `127.0.0.1:2080` (это debug mixed proxy
sing-box, который уже работает после основного деплоя). Прокси направляет трафик
через WireGuard-туннель на Foreign-узел:

```
bot ──SOCKS5──▶ sing-box proxy (127.0.0.1:2080) ──WireGuard──▶ Foreign ──▶ Telegram API
```

### Как развернуть бота

Ты можешь развернуть бота двумя способами:

**Способ 1: через агента.** Попроси агента в терминале:

> Разверни Telegram-бота. Токен: 123456:ABC-DEF... Мой телефон: +79161234567, роль: admin.

Агент создаст код бота, загрузит на RU-сервер, установит зависимости
(`python-telegram-bot`, `PyYAML`, `qrcode`, `Pillow`, `PySocks`), настроит
systemd-сервис, ACL и запустит.

**Способ 2: через автора.** Попроси автора оригинального проекта скинуть полный
код бота (директория `bot/`). После этого скажи агенту:

> Вот код бота в папке bot/. Разверни его на RU-сервере. Токен: ... Телефон: ...

### Первый запуск и настройка ACL

1. Создай бота через [@BotFather](https://t.me/BotFather):
   - Напиши `/newbot`, придумай имя и username
   - Скопируй токен (выглядит как `123456:ABC-DEF...`)
2. Открой бота, нажми `/start`, поделись контактом
3. Бот проверит твой номер телефона по `config/bot_acl.yaml` и выдаст меню
4. ACL настраивается в `config/bot_acl.yaml`:

```yaml
admins:
  - phone: "+79161234567"
    name: "Ты"
users:
  - phone: "+79169876543"
    name: "Жена"
    max_keys: 5
```

После изменения ACL скажи агенту «задеплой бота».

bash scripts/health_check.sh && bash scripts/smoke_check.sh

2. Удали папку `~/myownvpn` и начни с Шага 3 заново.

## Важные правила

- **Не меняй** сгенерированные файлы в папке `build/` руками. Всегда говори агенту «задеплой», и он сам всё перегенерирует.
- **Не теряй** папку `inventory/` — в ней ключи и пароли. Скопируй проект на флешку или в зашифрованный архив.
- **Не показывай** никому содержимое `inventory/*.env` и `build/clients/*` — это твои приватные ключи.
- **Не запускай** `--refresh-secrets` без крайней необходимости — это сбросит все ключи, и все клиенты перестанут работать.
